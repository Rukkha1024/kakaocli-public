# 이슈 001: TDD 우선으로 KakaoTalk 로그아웃 명령 추가

**상태**: 완료
**생성일**: 2026-03-29

## 배경

현재 CLI는 로그인 관련 UI 상태를 감지할 수 있지만 KakaoTalk를 실제로 로그아웃시키지는 못합니다.
저장된 Keychain 자격증명은 유지하면서 실행 중인 세션만 종료하는 `kakaocli logout`
명령이 필요하며, 구현은 테스트 우선 방식으로 진행해야 합니다.

## 완료 기준

- [x] `kakaocli logout`가 최상위 명령으로 제공된다.
- [x] 명령이 저장된 자격증명을 삭제하지 않고 실행 중인 KakaoTalk 세션을 종료한다.
- [x] 이미 로그아웃된 상태면 성공으로 처리한다.
- [x] 앱이 꺼져 있으면 KakaoTalk를 새로 띄우지 않고 성공으로 처리한다.
- [x] 로그아웃 판단 로직과 로그인 화면 성공 판정을 단위 테스트로 검증한다.

## 작업 목록

- [x] 1. 로그아웃 판단/파싱 동작을 검증하는 Swift 테스트를 추가한다.
- [x] 2. `KakaoCore`에 테스트 가능한 로그아웃 자동화 seam을 구현한다.
- [x] 3. `logout` CLI 명령을 추가하고 루트 명령 목록에 등록한다.
- [x] 4. 관련 없는 README 변경을 건드리지 않고 작업 추적 문서와 CLI help에 명령 동작을 반영한다.
- [x] 5. 테스트와 수동 검증으로 로그아웃 동작을 확인한다.

## 참고 사항

- KakaoTalk 상태바 메뉴는 앱이 이미 로그인 화면으로 돌아온 뒤에도 잠시 `Log out`를 계속 보여줄 수 있어서, 실제 성공 판정은 로그인 창 노출 여부를 기준으로 잡았다.
